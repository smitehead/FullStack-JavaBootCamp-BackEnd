## 2. 안 된 부분 (미구현 및 오류 상태 유지 중)

기존 28일자 피드백 중 여전히 해결되지 않고 남아있는 항목들입니다.

- **해더 (Header & Points)**
  - 포인트 입/출금이 동작하지 않음.
  - 내역 갱신이 안 되며, 전체 기간 필터 사용 시 연도 구분이 안 되어 확인 불가.
  - 찜하기 누를 시 카운트 증가가 즉시 동기화되지 않음.
  - 카테고리 대분류 클릭 시 이동하지 않음.
- **판매자 페이지 (`/seller/147`)**
  - 판매자 프로필 사진 변경 및 데이터(가입일, 상품 수, 후기 등) 연동 미비.
  - 차단된 사용자가 여전히 노출되는 버그 (UI 필터링 미적용).
- **마이페이지 (`/mypage`)**
  - 입찰/구매/찜 목록 수치가 수동(버튼 클릭)으로만 갱신됨.
  - 가입일이 무조건 현재 날짜로 표기되는 버그.
- **상품 상세 및 경매 등록 (`/products/98`, `/register`)**
  - 상품 문의 추가/수정/삭제 미구현.
  - 경매 주소 입력 시 정규식 조건 누락 및 "내 주소 불러오기" 부재.
  - 상품 이미지 중복 추가 허용 버그. 비어있는 재게시 폼 주소 입력란 부재.
  - 배송비가 3,000원으로 고정되어 있는 문제.
- **사용자 설정 및 알림 (`/settings`, `/inbox`)**
  - 채팅 및 알림 기능 미구현.
  - 알림 설정 ON/OFF 값이 저장되지 않고 원상 복구됨.
  - 프로필 수정(닉네임, 번호, 이메일, 주소) 미작동 방치. 번호 입력 힌트(placeholder) 누락.
  - 차단 사용자 추가/해제 시 변경 사항이 즉시 화면에 반영되지 않음.
- **검색 (`/search`)**
  - 대분류 필터링 미작동(중분류만 작동) 및 지역 필터 시 목록이 아예 뜨지 않음.

## 3. 보강 해야할 부분

- **프론트엔드 렌더링 및 상태(State) 관리 강화**
  - 포인트 내역, 찜 카운트, 마이페이지 숫자 지표, 프로필 변경 사항 등 **대부분의 이슈가 "클릭이나 새로고침을 해야만 바뀐다"는 상태 갱신 문제에 기인**합니다. 
  - 백엔드에 API를 요청한 후, 응답이 성공하면 React 내 전역 상태나 로컬 상태를 즉각 업데이트하는(Optimistic UI 업데이트 등) 데이터 플로우 최적화가 강력히 요구됩니다.
- **유효성 검사 (Validation)**
  - 이메일, 주소 정규식, 프로필 입력값 제어, 중복 이미지 업로드 방지 등 폼(Form)에서 사용자의 이상 행동을 막는 클라이언트 측 예외 처리의 꼼꼼한 보완이 필요합니다.
- **API 연동 안정성 확인 (알림 및 설정)**
  - 설정 ON/OFF, 차단 해제 등의 토글 액션이 DB에 영속적으로 반영되는지 백엔드 컨트롤러와 프론트엔드 API 호출부를 교차 점검해야 합니다.

## 4. 추가 했으면 좋았을 부분

- **공통 Custom Hook 설계 도입**
  - 찜하기, 차단하기, 토스트 알림 등 프로젝트 전반에 쓰이는 기능들을 개별 페이지에 흩어 작성하지 않고 공통 훅(예: `useHeart`, `useUserBlock`)으로 구성했다면 갱신 누락 버그 개선이 훨씬 쉬웠을 것입니다.
- **실시간 소켓 / SSE (Server-Sent Events) 도입**
  - 경매 사이트의 핵심은 실시간성입니다. 특히 채팅장(`inbox`)이나 자동 입찰 시점의 포인트 변동 알림을 위해 웹소켓을 초기에 도입하는 아키텍처 결정을 했으면 더 역동적이고 프리미엄한 웹앱이 구현되었을 것입니다. `sonner` 라이브러리를 통해 실시간 알림을 띄우는 방향으로 발전시키길 권장합니다.
- **배송비/지역/카테고리 관련 스키마 정교화**
  - 고정된 배송비, 검색 조건 미반영 등은 DB와 API 모델 설계 시 쿼리 파라미터나 상태 조건 매핑이 덜 정교해서 발생합니다. 필터링 API 설계안을 한 번 더 논의하여 백엔드 동적 쿼리(QueryDSL 등)를 정리하는 것이 좋습니다.
